## 🛡️ 5중 리스크 방어선

이 시스템의 핵심 차별점입니다. AI가 잘못 판단해도 자산이 보호됩니다.

```
주문 생성
   │
   ▼
┌──────────────────────────────────────────────────┐
│  1층  Kill Switch (규칙)              ⏱️ 0.01초  │
│       거래횟수 50회/일 · 회전율 30% · 단일주문 10%  │
│       → 숫자가 이상하면 즉시 차단                    │
├──────────────────────────────────────────────────┤
│  2층  LLM 사전검증 (AI 검문소)        ⏱️ 1~3초   │
│       논리일관성 · 이벤트타이밍 · 뉴스충돌           │
│       이상패턴 · 포트폴리오정합성                     │
│       → 맥락이 이상하면 거부/축소/보류               │
├──────────────────────────────────────────────────┤
│  3층  진입 필터 (규칙)                ⏱️ 0.01초  │
│       VIX>35 차단 · 크레딧 90% 차단 · 집중도 제한   │
│       → 시장이 위험하면 매수 차단                    │
├──────────────────────────────────────────────────┤
│  4층  실시간 경고 레벨                ⏱️ 상시    │
│       🟢정상 → 🟡주의 → 🟠경고 → 🔴비상            │
│       → 단계별 자동 포지션 축소                      │
├──────────────────────────────────────────────────┤
│  5층  기계적 손절 (최후 안전장치)     ⏱️ 상시     │
│       종목 -7% · 포트 일일-3% · 주간-5% · 월간-10%  │
│       → AI 판단과 무관하게 무조건 실행               │
│       → 월간 -10% 시 전량 현금화 + 30일 냉각        │
└──────────────────────────────────────────────────┘
   │
   ▼
주문 실행 (승인된 것만)
```

> **손절과 비상 매도는 1~2층을 건너뛰고 즉시 실행됩니다.**
> Kill Switch는 LLM이 장애여도 독립적으로 작동합니다.

## 🧠 LLM 분석 엔진

| 분석 유형 | 모델 | 실행 주기 | 역할 |
|-----------|------|-----------|------|
| 시장 레짐 판단 | Gemini Pro | 일 1회 | 확장/둔화/경고/위기 분류 → 자산배분 결정 |
| 뉴스 동향 분석 | Gemini Flash | 일 1회 | 섹터별 자금 흐름, 톤 변화 감지 |
| 위험 신호 감지 | Gemini Flash | 일 1회 + 긴급 | VIX, 크레딧 스프레드 이상 패턴 |
| 거래 사전검증 | Gemini Flash | 매 거래 전 | 5가지 맥락 검사 (AI 검문소) |
| 뉴스 감성분석 | Gemini Flash-Lite | 상시 배치 | 개별 뉴스 -1~+1 점수 |
| 사후 리뷰 | Gemini Flash | 일 1회 | 당일 거래 적절성 복기 |

**할루시네이션 방어:**
- LLM 출력을 규칙으로 교차 검증 (VIX 35+ 인데 확장기 판단? → 거부)
- 자산배분 합계 100% 검증, 비율 범위 검증
- 검증 실패 시 이전 유효 판단 자동 사용
- 모든 프롬프트·입력·출력을 DB에 기록 (재현성 확보)

**월 비용: ~₩10,000** (Gemini 무료 tier + 유료 시에도 극소액)

## 🧪 안전장치 검증 시스템 (Phase 0)

실전 투자 전에 모의투자로 전체 시스템을 검증합니다.

### Fault Injection — 이상거래 주입 테스트
매일 **의도적으로 이상한 주문**을 시스템에 주입하여 AI가 잡아내는지 확인합니다.

```
16개 시나리오 예시:
  ❌ 둔화기에 레버리지 ETF 대량 매수 → LLM이 거부해야 함
  ❌ FOMC 당일 대규모 포지션 변경 → LLM이 보류해야 함
  ❌ 부정 뉴스 직후 해당 섹터 매수 → LLM이 거부해야 함
  ✅ 정상 리밸런싱 → LLM이 승인해야 함 (과잉 차단 방지)
```

### Crisis Replay — 과거 위기 재현
코로나 폭락, 2022 금리 인상 등 과거 위기 데이터를 시스템에 주입하여 올바르게 방어하는지 검증합니다.

### Daily Drill — 매일 아침 자동 훈련
장 시작 30분 전(08:30) 자동 실행. 결과를 텔레그램으로 리포팅합니다.

### 실전 전환 조건
```
✅ 모의투자 30일 이상 무장애 운영
✅ 아침 훈련 통과율 85% 이상
✅ False Negative(놓친 이상거래) 14일 연속 0건
✅ 준비도 점수 80점/100점 이상
✅ Kill Switch 의도적 트리거 3회 통과
✅ 과거 위기 리플레이 3개 시나리오 통과
```

## 🛠️ 기술 스택

| 구분 | 기술 | 선택 이유 |
|------|------|-----------|
| **언어** | Python 3.11+ | AI/금융 생태계, async 지원 |

| **스케줄러** | APScheduler | Python 네이티브, cron/interval |
| **알림** | Telegram Bot API | 실시간 매매/경고/리포트 |
| **지표 계산** | ta (Technical Analysis) | 순수 Python, ta-lib 대안 |

```

┌──────────────────────────────────────────────────────────┐

│       │                                    │              │
│       │              RiskService ◄── PortfolioService     │

│                          │                                │
│                    MonitoringService                       │
└──────────────────────────────────────────────────────────┘

  함수 호출 → HTTP 호출로 래퍼만 교체 (인터페이스가 이미 명확)
```
